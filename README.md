# Homeless Builder

**Daniel Rickey** — AI systems builder, San Diego.

I'm building a real AI services business while figuring out how to do it. These repositories document the tools, infrastructure, and methodology being developed along the way — built in public, refined in practice, and released as open-source where it makes sense.

---

## The Work

### [devlog-engine](https://github.com/HomelessBuilder/devlog-engine)
A local MySQL backend and write script for federated developer journals. The infrastructure layer of the recursive development ecosystem — built with the system, for the system.

### [claude-journal](https://github.com/HomelessBuilder/claude-journal)
A flat-file developer journal covering all machine activity. Re-entry aid and content pipeline. The prototype that devlog-engine will eventually replace.

### [meta-system](https://github.com/HomelessBuilder/meta-system)
Context files and architecture documentation for the recursive development ecosystem. How the whole thing is organized and why.

### [monitor](https://github.com/HomelessBuilder/monitor)
A passive observer for the development ecosystem. Runs hourly, checks activity across all projects, flags concerns. Invisible to the system it watches.

---

## The Idea

Most development tooling is built for teams and enterprises. This is an experiment in building a self-improving, recursive development environment for a solo operator — using AI tools to punch above weight class.

The tools built here are used to build the next tools. The journals feed the course. The course funds the next project. Everything connects.

The brand documents the journey honestly — including the parts that don't look good yet. If you're building something real with limited resources and AI tools, this is the work that shows it's possible.
